Construction PH-Map on a multiple task learning framework
Framework Design of PH-Map. PH-Map employs a multi-task learning (MTL) framework with a hard parameter-sharing strategy to simultaneously predict multiple hierarchical levels of cell type annotation. All classification tasks share a common feature extraction backbone, while maintaining separate task-specific output layers for each hierarchical level. The shared backbone consists of a feedforward neural network with two fully connected hidden layers (200 and 100 neurons, respectively), utilizing ReLU activations and dropout regularization (rate=0.4). For the PH atlas, four independent classification heads are configured to predict the respective classes at each hierarchical level (Level 1 to Level 4). The model is trained by minimizing a weighted sum of cross-entropy losses across all levels, with weights increasing for finer-grained annotations to balance task learning. Optimization is performed using the Adam optimizer (learning rate=0.001) via backpropagation, which jointly updates both shared and task-specific parameters, thereby learning a feature representation that benefits all hierarchical classification tasks simultaneously.

Model training using PH-Map. PH-Map provides a reproducible training pipeline for constructing hierarchical cell-type classifiers from user-supplied reference single-cell transcriptomics data. The pipeline accepts a preprocessed AnnData object containing a binarized gene expression matrix (AnnData.X) and one or multiple columns of cell-type annotations (AnnData.obs). At its core is a hard parameter-sharing multi-task neural network architecture: a shared feature extraction backbone (defaulting to two fully connected layers with ReLU activation and Dropout regularization) connects to multiple task-specific classification heads, each corresponding to a target annotation level. Users have full flexibility to define the training tasks, including the number of hierarchical levels, the column names for each level, and the assignment of custom loss weights (task_weights) to balance the model's focus across different granularities. The model is trained by minimizing a weighted sum of cross-entropy losses across all levels. All parameters are jointly optimized using the Adam optimizer, with early stopping based on validation performance to prevent overfitting. Upon completion, the model weights, label encoders, training gene list, and other metadata are packaged into a portable TrainedModel object, ready for automated hierarchical annotation of new query datasets. The reference model in this study serves as an illustrative instance of this framework, trained on a specific pulmonary hypertension lung reference atlas with a customized four-level annotation schema and corresponding weights ([0.3, 0.8, 1.5, 2.0]).

Hierarchical annotation using PH-Map. PH-Map employs a standardized workflow for hierarchical cell type annotation on query single-cell or spatial transcriptomics data. Input data are required as AnnData objects containing a gene expression matrix. A critical gene alignment step ensures compatibility by mapping query genes to the model's training feature space, padding missing genes with zeros and binarizing expression values. Annotation is performed via the phmap.predict() function, which utilizes the pre-trained multi-task neural network. This network simultaneously outputs probability distributions across all hierarchical levels for each cell. The discrete predicted labels and corresponding confidence scores are decoded and returned in a structured DataFrame. These results can be seamlessly integrated back into the original AnnData object for downstream analysis. Additionally, PH-Map provides visualization tools, such as Sankey diagrams and probability plots, to interpret the hierarchical predictions and assess annotation confidence.

Benchmarking Model Training and Prediction Performance
Benchmark for Model Training Speed. To evaluate the computational efficiency of model construction, we compared the training times of PH-Map, CellTypist, and scANVI across a gradient of sample sizes (n = 1k, 3k, 6k, 10k, 15k, 20k, 50k, 100k) derived from a common PH-specific reference atlas. All methods were trained on the same series of downsampled datasets, annotated with a consistent four-tier cellular hierarchy. PH-Map was trained once in a multi-task learning framework to jointly predict all four annotation levels. In contrast, four independent CellTypist models were trained sequentially, one for each annotation tier. For scANVI, due to the excessively long training time which diminishes the significance of comparison, only a semi-supervised model was trained using the most refined labels (Level 4). To evaluate scalability, the total wall-clock training time of each method was recorded for different sample sizes.

Benchmark for Model Prediction Performance. We assessed the inference performance of models trained on the 50k-cell subset. Their prediction speed and annotation accuracy were evaluated on separate test datasets from the same atlas (n = 1k to 20k). PH-Map performed a single forward pass to simultaneously predict all four hierarchical labels. CellTypist required four sequential prediction calls, one per tier. The scANVI prediction involved an additional query-tuning step. For each method and test size, we recorded the total prediction time. Accuracy was calculated by comparing the predicted labels against the held-out ground truth for each annotation tier (for scANVI, only Level 4), providing a dual perspective on practical utility.
